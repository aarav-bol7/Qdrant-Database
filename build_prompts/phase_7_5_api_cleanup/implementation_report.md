## build_payload signature change — confirmation

```
$ uv run python -c "
from inspect import signature
from apps.ingestion.payload import build_payload, ScrapedItem, ScrapedSource
sig = signature(build_payload)
print('build_payload params:', list(sig.parameters))
print('ScrapedItem fields:', list(ScrapedItem.__dataclass_fields__))
print('ScrapedSource fields:', list(ScrapedSource.__dataclass_fields__))
"
build_payload params: ['chunk', 'tenant_id', 'bot_id', 'doc_id', 'item', 'source', 'uploaded_at']
ScrapedItem fields: ['item_index', 'section_path', 'page_number']
ScrapedSource fields: ['type', 'filename', 'url', 'content_hash']
```

`custom_metadata` kwarg removed. `pipeline.py` updated atomically: the `ScrapedItem` constructor no longer takes `item_type`/`title`/`url`/`language`; `ScrapedSource` no longer takes `language`; `auto_idx` is assigned via `enumerate(items_data)` and used as `item_index` everywhere (no fallback to `item_data["item_index"]`).

The `flat` list is now a 3-tuple `(auto_idx, item_data, c)` and the texts unpacking became `[c.text for _, _, c in flat]` (subtle one-liner that the test suite caught immediately when 5/5 pipeline tests went red until the unpack matched the 3-tuple shape — fixed in the same Phase C step).

## Backward compatibility check

The `apps.qdrant_core.search.search()` function is unchanged in Phase 7.5; it still passes through the full payload dict with whatever keys are present. Old chunks (uploaded with the Phase 5/7 schema) keep their `section_title`/`category`/`tags` payload keys. New chunks have the slim 17-key payload. Both flow through gRPC's response builder identically: the handler reads `chunk_dict.get("section_path")`, `chunk_dict.get("page_number")`, `chunk_dict.get("score")` only — old chunks' extra keys are silently dropped during Chunk message construction.

A direct Qdrant insertion test (manually `client.upsert(...)` an old-shape payload, then HTTP search to confirm response is valid) was deferred — it requires loading BGE-M3 to embed a query, which the host's `/app/.cache/bge` permission blocks (same root cause as the 21 test skips). Plan §6 ambiguity A8 mentioned this as defensive; not blocking. Phase 8 ship gate can re-add it.
